# 사내 연차 관리 도구 — 설계 문서

작성일: 2026-06-30

## 1. 목적

회사 직원들의 연차(연차·월차)를 한 곳에서 관리한다. 직원 등록, 연차 사용
기록, 잔여 연차 자동 계산, 달력 조회, 반기별 보고서 엑셀 출력을 제공한다.

## 2. 기술 구조

- **로컬 Node 서버 + 정적 프론트엔드.** 사용자는 `시작.bat`을 더블클릭하면
  Node 서버가 켜지고 브라우저가 `http://localhost:<port>`로 자동으로 열린다.
- **자동 저장:** 데이터 변경 시마다 프론트엔드가 서버에 저장 요청을 보내고,
  서버가 **HTML 파일과 같은 폴더의 `data.json`** 에 즉시 기록한다. (클릭 0번,
  영구 저장)
- **런타임:** Node.js. 외부 npm 의존성 없이 Node 내장 모듈(`http`, `fs`)만 사용.
- **엑셀 출력:** ExcelJS 브라우저 빌드(`exceljs.min.js`)를 로컬 vendor 파일로
  포함해 서버가 정적 서빙 → 인터넷 없이 클라이언트에서 색상·테두리·셀 병합이
  포함된 `.xlsx` 생성. (무료 SheetJS는 셀 스타일링 미지원이라 ExcelJS 사용)
- **계산 로직 분리:** 연차 부여·이월·잔액 계산을 `leave-logic.js` 순수 함수
  모듈로 분리(브라우저/Node 양쪽 동작). Node 테스트로 검증하고, 동시에 브라우저에
  정적 파일로 서빙한다.
- 외부 네트워크 요청 0 (CDN 미사용, 전부 localhost).
- **수동 백업(선택):** `data.json`을 그대로 복사하면 백업. 별도 내보내기/가져오기
  버튼도 제공.

## 3. 데이터 모델

```
Employee {
  id: string
  name: string            // 이름
  joinDate: string        // 입사일 (YYYY-MM-DD)
  initialBalance: number  // 첫 운영 회계연도 초기 잔액 (수동 입력)
  active: boolean         // 재직 여부 (퇴사 시 false)
  resignDate?: string     // 퇴사일 (선택)
}

Usage {
  id: string
  employeeId: string
  date: string            // 사용 날짜 (YYYY-MM-DD)
  days: number            // 1 | 0.5 | 0.25
  note?: string           // 메모 (선택)
}

Settings {
  baseFiscalYear: number  // 운영 시작 회계연도 (초기잔액이 적용되는 해)
}
```

## 4. 연차 부여 로직

회계연도(매년 1월 1일) 기준으로 부여하되, 퇴사 시에는 입사일 기준으로
별도 정산해 비교한다.

### 4.1 연간 부여량 계산 (회계연도 기준)

- **입사 1년 미만 (입사한 해):** 입사일로부터 1개월 개근 시마다 1일씩 적립,
  최대 11일. (월 단위 자동 적립)
- **입사 다음 회계연도부터:** 기본 15일.
- **가산:** 근속 3년 이상부터 매 2년마다 +1일.
  - 공식: `grant = 15 + floor((근속연수 − 1) / 2)`, 단 **최대 25일**.
  - 근속연수는 부여 기준일(해당 연도 1/1)에서 입사일까지의 만(滿) 연수.
  - 검증: 1년→15, 2년→15, 3년→16, 5년→17, 7년→18 … 21년→25(상한).
- **첫 해 정액:** 회계연도 첫 부여 시 비례 배분 없이 정액 15일로 부여(단순화).

### 4.2 초기 잔액 (첫 운영 회계연도)

- 시스템 도입 첫 회계연도(`Settings.baseFiscalYear`) **시작 시점에 이미 재직 중인**
  직원은 시작 잔액을 관리자가 **직접 입력**한다 (`Employee.initialBalance`).
- 첫 운영 회계연도 **도중·이후에 입사**한 직원은 초기잔액 수동 입력 대상이 아니며,
  4.1의 월 1일 적립 규칙으로 자동 계산한다(입사일이 `baseFiscalYear` 1/1보다 늦으면
  자동 처리). 판별: `joinDate`가 `baseFiscalYear` 시작 이전이면 수동, 이후면 자동.
- 이후 연도부터는 4.1·4.3 규칙으로 전부 자동 계산되며 수동 입력하지 않는다.

### 4.3 연도 전환 (이월 규칙)

매년 1월 1일 기준으로 잔액을 재설정한다.

- **이월 없음:** 전년도 미사용(양수) 잔액은 소멸한다.
- **당겨쓰기 차감:** 전년도 잔액이 **음수**면(다음 해 연차를 당겨 쓴 것),
  그 음수만큼 새해 부여량에서 자동 차감한다.
  - 새해 시작 잔액 = `당해 연도 부여량 + min(전년도 연말잔액, 0)`
- 연중 사용분이 부여량을 초과하면 잔액은 **음수로 표시**(당겨쓰기 허용).

### 4.4 잔여 연차 계산

```
특정 시점 잔액 = 해당 연도 시작 잔액 − (그 해 1/1 ~ 해당 시점까지 사용분 합)
```

### 4.5 퇴사 정산

- 퇴사 직원에 대해 **입사일 기준** 누적 부여·사용을 계산해 회계연도 기준
  결과와 나란히 비교 표시한다 (법정 정산용 참고 자료).

## 5. 화면 구성

탭(또는 페이지) 기반 단일 페이지 앱.

1. **대시보드** — 전 직원 카드/표: 이름 · 올해 부여 · 사용 · 잔여를 한눈에.
   음수 잔여는 강조 표시.
2. **연차 사용 입력** — 직원 선택 + 날짜 + 일수(1 / 0.5 / 0.25 버튼) + 메모 →
   저장 즉시 잔여 갱신. 최근 사용 기록 목록·삭제 가능.
3. **달력** — 월간 캘린더. 날짜 칸에 "누가 며칠 사용"을 직원별 색상으로 표시.
   월 이동.
4. **보고서** — 6번 양식의 매트릭스. 연도·반기(상/하) 선택, 화면 미리보기 +
   **엑셀(.xlsx) 출력**.
5. **직원 관리** — 직원 추가/수정/삭제, 입사일·첫해 초기잔액 설정, 퇴사 처리.
6. **퇴사 정산** — 직원 선택 시 입사일 기준 vs 회계연도 기준 비교 출력.

## 6. 보고서 / 엑셀 양식

두 번째 참조 이미지와 동일한 매트릭스 구조.

- **제목:** `YYYY년 N월말 기준 연/월차 내역` (기준 월말), 우상단 `작성일: YYYY-MM-DD`
- **헤더 행:**
  - `입사일` — 직원별 입사일
  - `이름` — 직원 이름
  - `N/N기준` — 기준일 시점 근속기간 (예: "14년 11월")
- **본문:** 각 월에 대해 두 줄 교차
  - `YY년N월` — 해당 월 시작 시점 잔액
  - `YY.0N월 사용분` — 해당 월 사용 합계 (밑줄 표시)
  - 잔액(N월) = 전월 잔액 − 전월 사용분
- **마지막 행:** `YY년12월31일` — 연말 예상 잔액(이후 사용 없음 가정).
- **반기 보고서:** 상반기 = 6월말 기준, 하반기 = 12월말 기준. 기준 월 이후는
  사용분 없이 잔액 유지로 표시.
- 색상·테두리·셀 병합·열 너비를 적용해 ExcelJS로 `.xlsx`를 생성한다.

## 7. 가정 및 결정 사항

- **운영 첫해는 2026년** (`Settings.baseFiscalYear = 2026`). 2026년 1/1 시점
  재직 직원의 시작 잔액은 관리자가 직접 입력한다.
- **2027년부터는 부여·차감 모두 자동.** 4.1(부여량)·4.3(당겨쓰기 차감) 규칙으로
  매년 1/1 자동 계산하며 수동 입력하지 않는다.
- 미래 연도의 자동 부여에서, 입사 1년 미만 신입은 월 1일 적립(4.1)으로 처리하며,
  입사 다음 회계연도 첫 부여는 정액 15일(전년 재직일 비례 미적용)로 단순화한다.
- 근속연수는 부여 기준일(1/1) 기준 만 연수.
- 데이터는 단일 사용자(관리자) PC의 `data.json` 파일에 서버가 자동 저장. 다중
  동시 접속은 범위 밖(로컬 단독 사용).

## 8. 범위 밖 (YAGNI)

- 다중 사용자 동시 접속·권한 관리
- 서버·DB·로그인
- 반차/시간 단위 외 사용자 정의 단위
- 연차 촉진 알림, 메일 발송
